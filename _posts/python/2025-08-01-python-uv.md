## uv란?

기존 파이썬 프로젝트는 **venv**를 이용한 가상 환경 생성 후 `pip` 명령으로 패키지를 설치하는 것이 일반적이었습니다. 이 과정은 때로 느리고 번거로웠으며, **requirements.txt** 파일을 수동으로 관리해야 하는 불편함이 있었습니다.

**uv**는 이러한 문제점을 해결하기 위해 등장한 차세대 패키지 관리 도구입니다. `pip`, `pip-tools`, `venv`의 기능을 통합하고, 압도적인 속도를 자랑하여 파이썬 개발 워크플로우를 혁신적으로 개선합니다.

즉 **uv**를 사용하면 프로젝트의 파이썬 버전을 명확히 지정하고 그에 맞는 의존성을 쉽게 관리할 수 있어 배포 시 발생할 수 있는 호환성 문제를 효과적으로 방지할 수 있습니다.

## uv 설치하기

**macOS 또는 Linux**

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
# 또는
brew install uv
```

**Windows**

```bash
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

**PyPI**

```bash
pip install uv
```

## 프로젝트 생성

```bash
uv init uvpy
cd uvpy
```

위 명령어를 통해 **uv**가 권장하는 폴더 구조와 **pyproject.toml** 파일을 자동으로 생성할 수 있습니다. 생성되는 주요 파일은 다음과 같습니다.

```bash
uvpy/
├── .python-version
├── .gitignore
├── pyproject.toml
├── main.py
└── README.md
```

- `.python-version`: 프로젝트에 사용할 파이썬 버전을 지정합니다.
- `pyproject.toml`: **PEP 621** 표준에 따라 프로젝트의 의존성과 메타데이터를 정의하는 핵심 파일입니다.

## 패키지 관리

`uv add` 명령어를 사용하면 패키지를 설치함과 동시에 `pyproject.toml` 파일에 자동으로 기록됩니다.

```bash
uv add 라이브러리명
uv add pandas
uv add wxPython==4.2.2
```

- 테스트 프레임워크와 같이 개발 시에만 필요한 패키지는 `--dev` 옵션을 사용하여 별도로 관리할 수 있습니다.
- 패키지를 삭제하려면 `add` 대신 `remove`를 입력하면 됩니다.

`uv export`를 사용하면 `pyproject.toml`의 의존성 정보를 **requirements.txt** 형식으로 내보낼 수 있습니다.

```bash
uv export -o requirements.txt
```

## pip로 설치하기

**uv**는 자체적인 `uv pip install` 명령어를 제공하지만, 이는 일반적인 패키지 설치와는 용도가 다릅니다. 이 명령어로 설치한 패키지는 `pyproject.toml`에 기록되지 않기 때문에, 프로젝트의 공식적인 의존성으로 간주되지 않습니다. 따라서 일시적인 테스트나 스크립트 실행 등 임시적인 목적으로 사용하기에 적합합니다.

```bash
uv pip install numpy
```

## 스크립트 실행하기

```bash
uv run main.py
```

## Python 버전 관리

여러개의 파이썬 버전 설치:

```bash
uv python install 3.10 3.11 3.12
```

가상환경 생성:

```bash
uv venv --python 3.12.0
```
