## uv란?

**uv**는 **차세대 패키지/프로젝트 관리 도구**로, 기존의 `pip`, `venv` 같은 기능을 하나로 통합하면서 **속도와 간편함**을 강조한 최신 툴이에요.

즉 **uv**를 사용하면 프로젝트의 파이썬 버전을 명확히 지정하고 그에 맞는 의존성을 쉽게 관리할 수 있어 배포 시 발생할 수 있는 호환성 문제를 효과적으로 방지할 수 있습니다.

## uv 설치하기

**macOS 또는 Linux**

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
# 또는
brew install uv
```

**Windows**

```bash
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

**PyPI**

```bash
pip install uv
```

## 프로젝트 생성

```bash
uv init uvpy
cd uvpy
```

위 명령어를 통해 **uv**가 권장하는 폴더 구조와 **pyproject.toml** 파일을 자동으로 생성할 수 있습니다. 생성되는 주요 파일은 다음과 같습니다.

```bash
uvpy/
├── .python-version
├── .gitignore
├── pyproject.toml
├── main.py
└── README.md
```

- `.python-version`: 프로젝트에 사용할 파이썬 버전을 지정합니다.
- `pyproject.toml`: **PEP 621** 표준에 따라 프로젝트의 의존성과 메타데이터를 정의하는 핵심 파일입니다.

> **uv** 명령에서 입력한 `uvpy`는 프로젝트 폴더 이름을 의미하므로 원하는 이름으로 지정하시면 되요.
{: .notice--primary}

## 패키지 관리

`uv add` 명령어를 사용하면 패키지를 설치함과 동시에 `pyproject.toml` 파일에 자동으로 기록됩니다.

```bash
uv add 라이브러리명
uv add pandas
uv add wxPython==4.2.2
```

- 테스트 프레임워크와 같이 개발 시에만 필요한 패키지는 `--dev` 옵션을 사용하여 별도로 관리할 수 있습니다.
- 패키지를 삭제하려면 `add` 대신 `remove`를 입력하면 됩니다.

`uv export`를 사용하면 `pyproject.toml`의 의존성 정보를 **requirements.txt** 형식으로 내보낼 수 있습니다.

```bash
uv export -o requirements.txt
```

## pip로 설치하기

**uv**는 자체적인 `uv pip install` 명령어를 제공하지만, 이는 일반적인 패키지 설치와는 용도가 다릅니다. 이 명령어로 설치한 패키지는 `pyproject.toml`에 기록되지 않기 때문에, 프로젝트의 공식적인 의존성으로 간주되지 않습니다. 따라서 일시적인 테스트나 스크립트 실행 등 임시적인 목적으로 사용하기에 적합합니다.

```bash
uv pip install numpy
```

## 스크립트 실행하기

```bash
uv run main.py
```

## Python 버전 관리

여러개의 파이썬 버전 설치:

```bash
uv python install 3.10 3.11 3.12
```

가상환경 생성:

```bash
uv venv --python 3.12.0
```

> **Note**:
> **uv**를 사용하면서 몇 가지 문제가 있었는데요.
> 1. **VSCode**로 코드 작성 시 터미널이 **가상 환경**으로 접속하게 되는데 코드를 실행하다 보니 **uv** 패키가 설치되어 있지 않아 오류가 발생한다.
> 2. 가상환경에 pip 패키지가 없다.
> 3. 정리해서 말하자면 파이썬 루트 경로에 uv를 설치하고 사용을 하는데 vscode에서는 가상환경으로 접속을 하니 uv 패키지가 없어 오류가 나는 것이다. 또 기존에는 pip 명령어를 주로 이용했는데 uv로 가상환경을 생성하면 pip이 설치되지 않아서 `pip list`와 같은 명령어를 사용할 수 없음.
> 4. .ipynb에서는 uv로 설치 시 바로 반영이 되지 않는 증상이 있음.

> **Note**:  
> Python 프로젝트에서 **uv**를 사용하면서 다음과 같은 문제를 경험했다.  
> 1. **VSCode 터미널 가상환경 문제**:  
>   VSCode에서 코드를 실행하면 자동으로 **가상환경(venv)** 이 활성화되는데, 이 환경에는 기본적으로 **uv가 설치되어 있지 않아** `uv` 명령어를 실행하려면 가상환경을 빠져 나오거나 별도의 터미널을 호출해야 하는 불편이 있다.
> 1. **uv 기반 가상환경의 pip 미포함**:  
>   **uv**로 생성한 가상환경에서 설치된 패키지를 확인하기 위해 `pip list` 또는 `pip show` 명령어를 사용하려고 했으나 **pip 패키지가 설치되어 있지 않아** 사용할 수 없었다.
> 1. **Jupyter Notebook(.ipynb) 사용 문제**:
>   **uv**를 통해 패키지를 설치하면 **Jupyter Notebook 커널에는 즉시 반영되지 않는** 현상이 있어 다시 실행해야 하는 불편이 있었다.
