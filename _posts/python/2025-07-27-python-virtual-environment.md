## 파이썬 가상 환경, 왜 필요할까요?

프로젝트를 진행하다 보면 여러 **모듈**을 설치하게 되는데요, 이때 각 모듈은 서로 **의존성(Dependency)**을 가지는 경우가 많습니다. 만약 모듈을 **글로벌 환경**에 설치하면 여러 프로젝트를 관리하기 어려울 수 있습니다. 왜 그럴까요?

예를 들어, 프로젝트1에서는 모듈 A와 B를, 프로젝트2에서는 모듈 A와 C를 사용하는 상황을 생각해봅시다. 여기서 모듈 B는 A가 1.x 버전일 때, 모듈 C는 A가 2.x 버전일 때만 사용 가능하다고 해보죠.

만약 모듈을 글로벌 환경에 설치한다면 A는 1.x 또는 2.x 둘 중 하나만 설치할 수밖에 없습니다. 결국 두 프로젝트를 동시에 관리할 수 없게 되는 거죠. 이런 문제를 해결하기 위해 등장한 것이 바로 **파이썬 가상 환경**입니다.

가상 환경은 각각의 프로젝트에 필요한 모듈을 별도의 **로컬 환경**에 설치하여 사용하게 해줍니다. 마치 프로젝트마다 독립적인 공간을 만들어 주는 것과 같죠.

파이썬은 일반적으로 **pip**를 이용해 모듈을 설치하는데요, 이는 **npm**과 유사하지만 중요한 차이점이 있습니다. npm은 자동으로 로컬 환경에 모듈을 설치하는 반면, pip는 기본적으로 글로벌 환경에 설치합니다. 이 때문에 파이썬에서는 별도로 가상 환경이라는 개념을 사용하게 된 것입니다.

## 파이썬 가상 환경의 종류

앞서 말씀드렸듯이, 파이썬 **표준 라이브러리**에서 지원하는 가상 환경이 있고 그렇지 않은 것이 있습니다. 이 글에서는 두 가지 모두 소개해 드릴게요. 표준 라이브러리는 별도 설치 없이 사용할 수 있지만, 비표준 라이브러리는 따로 설치해야 합니다.

### 비표준 라이브러리 (별도 설치 필요)

1. **virtualenv**
   비표준 라이브러리 중 가장 많이 사용되는 가상 환경 라이브러리입니다. 표준 라이브러리는 아니지만, **PyPA** (파이썬 패키지 관리 권장 사항을 개발하고 유지 관리하는 비영리 단체)에서 공인한 라이브러리입니다. 아래에서 소개할 `venv`와 사용법이 유사하지만, Python 2와 3을 모두 지원한다는 장점 때문에 많이 사용됩니다. 서로 다른 파이썬 버전으로 개발해야 할 때 특히 유용하겠죠?

2. **pyenv**
   파이썬 버전별로 환경 생성이 가능한 라이브러리입니다. 가상 환경은 virtualenv 등을 통해 별도로 생성해야 합니다. 즉, 파이썬 버전 관리 기능만 제공한다고 생각할 수 있습니다. 아쉽게도 Windows는 지원하지 않습니다.

3. **pyenv-virtualenv**
   `pyenv`를 위한 플러그인입니다. `pyenv`와 `virtualenv`를 동시에 사용할 수 있도록 해줍니다.

4. **virtualenvwrapper**
   `virtualenv`에 대한 익스텐션 모음입니다. `virtualenv`를 사용할 때 번거로웠던 부분들을 편하게 사용할 수 있도록 돕는 도구입니다.

5. **pyenv-virtualenvwrapper**
   `pyenv`를 위한 플러그인입니다. `pyenv`와 `virtualenvwrapper`를 편하게 통합하여 사용할 수 있도록 해줍니다.

6. **pipenv**
   `Pipfile` (프로젝트의 의존성을 관리하는 파일), `pip`, `virtualenv`를 하나의 명령어로 실행할 수 있도록 돕는 라이브러리입니다. `pip` 명령어 대신 `pipenv` 명령어로 모듈을 설치하면 루트 디렉토리에 가상 환경이 생성됩니다. 실행할 때도 `pipenv` 명령어를 사용하여 가상 환경에서 작업을 수행합니다.

### 표준 라이브러리 (별도 설치 불필요)

1. **pyvenv**
   `venv`와 거의 유사한 가상 환경 라이브러리입니다. Python 3.3에 추가되었으나, Python 3.8에서 제거되었습니다.

2. **venv**
   Python 3.3부터 사용 가능한 가상 환경 라이브러리입니다. 현재 기준으로 유일하게 표준 라이브러리에서 지원하는 가상 환경 라이브러리입니다. 앞서 소개한 `virtualenv`의 경량화 모듈로 생각할 수 있으며, 실제로 두 라이브러리는 상당히 유사합니다. `venv`는 Python 2.x가 지원되지 않는 등, `virtualenv`보다 기능적으로는 조금 부족하지만, 별도 설치 없이 표준 라이브러리로 제공된다는 점에서 많이 사용됩니다.

그 외에도 **아나콘다**를 통해 가상 환경을 구축하는 등 여러 가지 방법이 있습니다. 일반적으로는 위에서 소개한 `virtualenv`나 `venv`를 통해 필요한 작업을 수행할 수 있으니, 추가로 궁금한 점은 직접 찾아보시는 것을 권해드립니다.

## venv 사용법

이제 **venv 사용법**에 대해 소개해 드릴게요. 모든 명령어는 프로젝트의 **루트 디렉토리** 터미널에서 실행해야 합니다.

### 가상 환경 생성

```vim
python -m venv 가상환경이름
python -m venv .venv
```

먼저 가상 환경을 생성하는 방법입니다. 위와 같이 가상 환경 이름을 명시해주고 실행합니다. 가상 환경이 잘 생성되었다면 루트 디렉토리에 해당 이름으로 폴더가 생성됩니다. 일반적으로 가상 환경 이름은 `.venv`로 정하는 것이 관례입니다.

```bash
┌─ venvTest
│  ├─ .venv
│  │  ├─ Include
│  │  ├─ Scripts
│  │  ├─ pyvenv.cfg
```

▲ 가상 환경 디렉토리 구성 예시

생성된 가상 환경 디렉토리(`.venv`)를 보면 `Include`, `Lib`, `Scripts`, `pyvenv.cfg`로 구성됩니다. 여기서 `Lib` 폴더에 우리가 설치한 모듈이 저장됩니다.

### 가상 환경 활성화

생성한 가상 환경을 사용하려면 활성화해야 합니다. 활성화할 가상 환경 이름을 명시하고 명령어를 실행합니다. 이렇게 이름을 명시한다는 것은 하나의 루트 디렉토리에 여러 개의 가상 환경을 만들고, 원하는 것을 활성화하여 프로그램을 실행할 수도 있다는 의미입니다.

1) **Windows (cmd)**

   ```bash
   call 가상환경이름/Scripts/activate
   ```

2) **Linux / macOS**

    ```bash
    source 가상환경이름/bin/activate
    ```

간혹 `activate` 명령어가 `Scripts` 폴더가 아닌 `bin` 폴더에 있는 경우도 있습니다. 생성된 가상 환경 폴더 구조를 잘 살펴보시고 상황에 맞게 작업하시면 됩니다.

또한, 다른 자료에서는 `source`로 실행하도록 설명되어 있는 경우가 많은데요, 윈도우 환경에서는 해당 명령어를 지원하지 않으니 `call` 명령어를 대신 사용하시면 됩니다.

활성화가 성공적으로 되면 위와 같이 커맨드 라인 좌측에 가상 환경 이름이 표시됩니다. 이제 이 가상 환경 안에서 작업을 할 수 있습니다.

### 가상 환경 모듈 설치

이제 가상 환경 내에 모듈을 설치해보도록 하겠습니다. 가상 환경 내에 설치될 뿐, 설치하는 방법이나 과정은 글로벌 환경에서 설치하는 것과 동일합니다.

예를 들어 `numpy` 모듈을 설치한다면 `pip install numpy`로 설치할 수 있습니다. 만약 프로젝트의 의존성 목록이 담긴 `requirements.txt` 파일이 있다면 `pip install -r requirements.txt` 명령어로 모듈을 한 번에 설치하시면 됩니다.

`Lib` 폴더에 `numpy` 모듈이 추가된 것을 확인할 수 있습니다.

### 가상 환경 비활성화

```vim
deactivate
```

가상 환경 사용을 마쳤다면 위의 명령어로 비활성화해줍니다. 이 명령을 수행하면 가상 환경을 빠져나와 일반 터미널로 돌아갑니다.

### 가상 환경 삭제

아나콘다를 사용하시던 분들은 가상 환경 삭제 명령어를 찾지 못해 혼란스러워하는 경우가 종종 있는데요, 파이썬 가상 환경을 삭제하고 싶다면 해당 폴더를 그냥 지우면 됩니다. 아주 간단하죠?

`virtualenv`도 `venv`와 거의 같은 방식으로 적용할 수 있으니 상황에 따라 적절히 사용하시길 바랍니다.
